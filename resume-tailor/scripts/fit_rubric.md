# Fit Assessment Rubric

Produce a candid, skeptical fit assessment BEFORE tailoring. The goal is accuracy, not
encouragement. Cover all five sections.

## 1. Hard gates (any failure = flag prominently, ask before proceeding)
- **Work authorization:** Does the JD say "no sponsorship"? Does the candidate need it?
  If unknown, ask. This overrides resume quality entirely.
- **Location / on-site:** Is it on-site or hybrid in a specific city? Will the candidate
  be there? If relocation is uncertain, flag it.
- **Seniority:** Does required years/level roughly match? Note over- or under-shooting.
- **Non-negotiable credentials:** Clearance, specific degree, licensure, etc.

## 2. Genuine strengths (map each to a JD line)
List strengths backed by real resume facts. For each, cite the JD requirement it satisfies. Prioritize hard requirements over nice-to-haves.

## 3. Real weaknesses / gaps
List what the candidate lacks relative to the JD — especially recurring themes that sit in
"nice to have." Be specific ("no MCP/LangChain/vector-search experience visible") rather
than vague. Distinguish:
- Gaps that are disqualifying vs. merely competitive disadvantages.
- Gaps the candidate could honestly close with a sentence of reframing vs. genuine voids.

## 4. Verdict & recommended action
One of: **strong fit** / **plausible fit** / **stretch** / **likely ineligible**. Then a
concrete recommended action — one of:
- **Apply now** — strong fit, no major gaps.
- **Apply with targeted cover letter** — plausible fit; cover letter should address specific gaps head-on.
- **Build project first, then apply** — stretch; the gap-closing project (section 5) would materially change the application.
- **Skip** — likely ineligible or hard-disqualified; time is better spent elsewhere.
One or two sentences justifying the recommendation. Never inflate the verdict to be encouraging.

## 5. Gap-closing project suggestion
Based on the gaps in section 3, propose one concrete side project that would close the
most critical ones. Requirements:
- **Specific:** Name the exact tools/frameworks from the JD the project should use.
- **Scoped:** Achievable in 1–2 weekends, not a month-long build.
- **Demonstrable:** Deployable or has a public repo an interviewer could look at.
- **Targeted:** Explain which gaps from section 3 it addresses.
- If the candidate is already a strong fit with no significant gaps, say so and skip.
